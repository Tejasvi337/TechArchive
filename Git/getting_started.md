# Git  

> **Git** is a distributed version control system (DVCS) that helps developers collaborate on projects, track changes, and manage code repositories efficiently. 

This guide will walk you through the essential steps to start using Git.

## **Topics**

### **Git Basics**

#### **1. Initializing Repositories**

- `git init`
- `git clone`
    - `git clone <url>`
    - `git clone <url> <new repo name>`

#### **2. Repository Status**

- `git status`
    - `git status --short`
    - `git status <path>`

####  **3. Tracking and Staging Changes**

- `git add`
    - `git add .`
    - `git add -A`

#### **4. Commit Changes**

- `git commit`
    - `git commit -a`
    - `git commit -m "<message>"`

#### **5. Viewing History**

- `git log`
    - `git log --oneline`
    - `git log -p`
    - `git log --stat`

#### **6. Viewing Changes**

- `git diff`
    - `git diff --staged`
    - `git diff <commit1> <commit2>`

#### **7. Removing Files**

- `git rm`
    - `git rm <file>`
    - `git rm --cached <file>`

#### **8. Moving and Renaming Files**

- `git mv <old_filename> <new_filename>`

#### **9. Ignoring Files**

- `.gitignore`
